# aws-sagemaker-role-fit

A serverless role-fit scoring engine powered by AWS SageMaker. Paste a job description and get a semantic match score against a candidate profile — broken down by skill domain — with an optional AI explanation generated by Amazon Bedrock.

[![AWS](https://img.shields.io/badge/AWS-SageMaker-FF9900?style=flat&logo=amazon-aws)](https://aws.amazon.com/sagemaker/)

## How it works

```
POST /match  { "job_description": "..." }
        │
        ▼
Lambda (match_handler.py)
        │
        ├── Chunk JD into overlapping 1,400-char segments (max 5)
        │   → solves the 512-token hard limit on all-MiniLM-L6-v2
        │
        ├── Embed each chunk via SageMaker Serverless Endpoint
        │   sentence-transformers/all-MiniLM-L6-v2
        │
        ├── Average-pool chunk embeddings → single JD vector
        │
        ├── Cosine similarity vs. pre-computed profile domain embeddings (S3)
        │
        ├── Store result in DynamoDB (analysis_id, scores, JD text, 30-day TTL)
        │
        └── { "analysis_id": "...", "overall_match": 87, "domains": { ... } }

POST /explain  { "analysis_id": "..." }   ← optional, Bedrock-powered
        │
        ▼
Lambda (handler.py)
        │
        ├── Fetch analysis + full JD text from DynamoDB
        ├── Call Amazon Bedrock (Claude Haiku) for qualitative analysis
        └── { "job_title": "...", "employer": "...", "summary": "...",
              "domains": [{ "name": "...", "relevant": true, "score": 87,
                            "alignment": "**Why this is solid**\n- ..." }] }

POST /pdf  { "overall_score": 87, "domains": [...], ... }
        │
        ▼
Lambda (handler.py)
        │
        ├── Generate PDF via fpdf2 + DejaVu Condensed TTF (Unicode)
        └── { "pdf": "<base64>" }   ← client decodes and triggers download

POST /feedback  { "analysis_id": "...", "accurate": true, "reasons": [...] }
        └── Stores feedback in DynamoDB (90-day TTL)
```

**No training data required** — zero-shot semantic similarity using a pre-trained sentence transformer.

## Architecture

| Component | Service |
|-----------|---------|
| Embedding model | SageMaker Serverless Inference — `all-MiniLM-L6-v2` via HuggingFace DLC |
| Chunking | Lambda — overlapping 1,400-char segments, average-pooled |
| Scoring logic | AWS Lambda (Python 3.12) |
| AI explanation | Amazon Bedrock (Claude Haiku) via `/explain` endpoint |
| PDF generation | AWS Lambda — fpdf2 + DejaVu Unicode font |
| API | Amazon API Gateway REST — `/match`, `/explain`, `/pdf`, `/feedback` |
| Profile embeddings | Amazon S3 (pre-computed, AES256 encrypted) |
| Analysis storage | Amazon DynamoDB (30-day TTL on analyses, 90-day on feedback) |
| IaC | Terraform |
| CI/CD | GitHub Actions (OIDC, no stored credentials) |

**SageMaker Serverless Inference** — scales to zero between requests, no idle compute cost.

## Chunking — handling the 512-token limit

`all-MiniLM-L6-v2` has a hard 512-token input limit. Long job descriptions are split into overlapping chunks before embedding:

```python
CHUNK_SIZE      = 1400   # ~400 tokens — safely under the 512-token limit
CHUNK_OVERLAP   = 150    # character overlap so boundary context isn't lost
MAX_CHUNKS      = 5      # cap to bound latency
MAX_INPUT_CHARS = 8000   # full JD accepted; chunking handles the rest
```

Each chunk is embedded independently, then all chunk vectors are **average-pooled** into a single JD vector before cosine similarity is computed. This gives full coverage of the job description without truncation.

---

## Scoring

```python
score = max(0, min(100, round(cosine_similarity * 165)))
```

Cosine similarity for sentence transformers compresses into ~0.3–0.7 even for strong matches. The multiplier maps this to a human-readable 0–100 range.

## Skill domains

Define your own domains in `scripts/compute_embeddings.py`. Each domain is a short text description — the embedder turns it into a vector for comparison.

Example (replace with your own):

| Domain key | What it covers |
|------------|---------------|
| `domain_1` | Cloud platforms and infrastructure |
| `domain_2` | Security, DevSecOps, compliance |
| `domain_3` | AI/ML, GenAI, data engineering |
| `domain_4` | APIs, integration, microservices |
| `domain_5` | Seniority and technical leadership |

Add or remove domains freely — the Lambda reads domain keys dynamically from the embeddings file in S3. After changing domains, re-run `compute_embeddings.py` and re-upload to S3. No Lambda redeployment needed.

## Deploy

### Prerequisites
- AWS account with SageMaker and Bedrock access
- Terraform >= 1.6
- S3 + DynamoDB for Terraform state (update `backend.tf`)
- GitHub OIDC role with permissions for SageMaker, Lambda, API Gateway, S3, DynamoDB, IAM, SSM

### 1 — Deploy infrastructure

```bash
cd infra
terraform init
terraform apply
```

### 2 — Compute and upload profile embeddings

```bash
pip install -r scripts/requirements.txt
python scripts/compute_embeddings.py <embeddings-bucket-name>
```

The bucket name is printed in Terraform outputs.

### 3 — Test the endpoint

```bash
curl -X POST $(terraform -chdir=infra output -raw api_endpoint)/match \
  -H "Content-Type: application/json" \
  -d '{"job_description": "Senior DevSecOps Engineer with AWS EKS and Terraform experience"}'
```

## API

### `POST /match`

**Request:**
```json
{ "job_description": "Job description text (up to 8,000 chars)" }
```

**Response:**
```json
{
  "analysis_id": "uuid",
  "overall_match": 87,
  "domains": {
    "domain_1": { "label": "Cloud platforms and infrastructure", "score": 92 },
    "domain_2": { "label": "Security, DevSecOps, compliance",    "score": 88 }
  }
}
```

---

### `POST /explain`

Calls Amazon Bedrock (Claude Haiku) for a qualitative analysis. Uses the full stored JD text from DynamoDB — no need to resend the job description.

**Request:**
```json
{ "analysis_id": "uuid-from-match-response" }
```

**Response:**
```json
{
  "job_title": "Senior DevSecOps Engineer",
  "employer": "Acme Corp",
  "summary": "Strong overall fit. Cloud and security domains align well...",
  "relevant_score": 89,
  "domains": [
    {
      "name": "Cloud platforms and infrastructure",
      "relevant": true,
      "score": 92,
      "alignment": "**Why this is solid**\n- Explicit EKS requirement matches experience\n**Why not higher**\n- No mention of multi-cloud"
    },
    {
      "name": "Data engineering",
      "relevant": false,
      "score": 41,
      "alignment": ""
    }
  ]
}
```

`relevant_score` is the overall match recalculated using only the domains marked `relevant: true`.

---

### `POST /pdf`

Generates a formatted PDF report. Returns base64-encoded PDF — decode and trigger browser download on the client.

**Request:**
```json
{
  "job_title": "Senior DevSecOps Engineer",
  "employer": "Acme Corp",
  "overall_score": 89,
  "has_note": true,
  "summary": "Strong overall fit...",
  "domains": [ ... ]
}
```

**Response:**
```json
{ "pdf": "<base64-encoded PDF>" }
```

---

### `POST /feedback`

**Request:**
```json
{
  "analysis_id": "uuid",
  "accurate": true,
  "reasons": ["Score feels too low"],
  "comment": "Optional free text"
}
```

**Response:**
```json
{ "ok": true }
```

## Lambda build (CI)

DejaVu fonts are downloaded explicitly from the dejavu-fonts GitHub releases — fpdf2 v2.7.0+ no longer bundles fonts.

```yaml
- name: Build Lambda packages
  run: |
    # match scorer
    pip install -t lambda_pkg/ --quiet
    cp lambda_src/match_handler.py lambda_pkg/
    cd lambda_pkg && zip -r ../lambda_match.zip . && cd ..
    rm -rf lambda_pkg

    # explain + pdf handler (needs fpdf2 + DejaVu fonts)
    mkdir -p lambda_pkg/fonts
    pip install pypdf fpdf2 -t lambda_pkg/ --quiet
    DEJAVU_VER=2.37
    curl -sL "https://github.com/dejavu-fonts/dejavu-fonts/releases/download/version_2_37/dejavu-fonts-ttf-2.37.tar.bz2" \
      | tar -xj --wildcards --strip-components=2 -C lambda_pkg/fonts \
        "*/ttf/DejaVuSansCondensed.ttf" "*/ttf/DejaVuSansCondensed-Bold.ttf"
    cp lambda_src/handler.py lambda_pkg/
    cd lambda_pkg && zip -r ../lambda_agent.zip . && cd ..
    rm -rf lambda_pkg
```

Fonts land at `/var/task/fonts/` in Lambda. The handler references that fixed path.
